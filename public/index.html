        How Deep Packet Inspection Works

        DeepInsight analyzes each network packet through multiple layers to identify its application origin and apply blocking policies.

        01
        Ethernet Parse

        Extracts source/destination MAC addresses and validates EtherType for IPv4 (0x0800).

        02
        IPv4 Header

        Reads source/destination IP, protocol field (TCP=6, UDP=17) and IHL for payload offset.

        03
        5-Tuple Flow Tracking

        Maps packets to bidirectional conversations via (SrcIP, DstIP, SrcPort, DstPort, Protocol).

        04
        TLS SNI Extraction

        Parses TLS Client Hello on port 443 to find Server Name Indication in plaintext — even HTTPS leaks the domain name here.

        05
        HTTP Host Parsing

        For port 80, scans payload for Host: header and extracts the target hostname.

        06
        Policy Engine

        Matches each flow against the rule set. Blocked flows are dropped from the output PCAP entirely.
